hexo部署问题记录

发表于 2017-02-21 |

转自：http://chasen-g.cn/

写在前面：这几天部署github + hexo真是遇到了不少坑。我真不知道为什么脑袋一热我就想用hexo了，
可能是因为hexo对于新手程序员的高大上和高逼格；也有可能是next主题给我的极简的视觉体验，好吧，
无论怎么样，最后还是比较成功的，下面总结一下我这几天遇到的坑。

安装

安装对于新手来说真的很恶心，不容易上手，网上的教程要不然就是写的不全面，要不然就全都是复制，或者百度百科
上一些不知所云的提问和回答，反正我是搞了一下午才搞明白。

step 1

无论是mac电脑还是pc电脑，想在电脑上安装hexo就必须先安装几个重要的东西：

1、node.js （http://nodejs.cn 上根据系统下载安装就可以 以下简称node）

2、github for mac 或者 github for windows （百度一搜就能下载了）

3、github的账户 （注册看不懂请用百度翻译，github没有中文版！）

4、hexo

node的安装很简单，pc电脑双击安装一直下一步就行，mac电脑使用pkg格式的安装包 也一样，双击安装。

安装完之后简称是否安装成功了，

pc：打开命令行，输入node -v ，如果提示了版本号，就是安装成功了。（安装不成功的情况还没有遇到过）
mac：打开终端，输入node -v

step 2

安装github，pc版我建议百度搜索一下github离线安装包，因为真的是慢的一比。

mac下比较快 安装的时候可能会提示“是否安装命令行”之类的提示，同意即可。

mac下安装完之后，可能会提示需要下载xcode什么之类的才能运行，放心下载就行了，都是自动下载安装的。

step 6

写博客

cd到hexo目录，执行hexo new post “博客名称”

然后使用markdown语法 修改source/_post里面的 .md文件即可，
这里推荐一个在线的markdown编辑器 http://mahua.jser.me

每次写完博客，都需要执行hexo g 和hexo d。

step 7

想用另一台电脑编辑博客怎么办？

没什么有效的办法，建议把hexo目录存放到u盘里，随时用随时查，但电脑必须安装了node。

如果我换了新电脑怎么办？

我建议你就把库删了，重新按照流程部署一个，然后把以前目录source/_post里面的日志拷贝过来，但我不知道有没有效。

我在网上搜索了一下答案，大概是库新建一个分支，静态页面放到master下，原始文件放到新分支下，每次写完日志git把原始文件提交到新分支，静态页面提交到master，但是我没弄明白。
